# Jack Compiler - Assignment 1 & 2

A systematic implementation of a **Jack Language Compiler** built with **Rust**. This tool performs lexical and syntactic analysis to transform Jack source code into standardized XML representations.
To explain the core functionality of a compiler, we look at the two distinct stages of the front-end architecture: the **Tokenizer** and the **Parser**.

### **The Tokenizer (Lexical Analysis)**
The Tokenizer is the first stage of the compilation process. It acts as a scanner that reads the raw stream of characters from a source file and groups them into meaningful units called **tokens**.

*   **Categorization**: It identifies and labels each unit as a specific type, such as a **Keyword** (`class`, `let`), **Symbol** (`{`, `+`), **IntegerConstant**, **StringConstant**, or **Identifier**.
*   **Cleaning**: It systematically removes whitespace and comments (both line and block comments) that are not necessary for the program's logic.
*   **Validation**: It performs basic checks, such as ensuring integer constants fall within the valid range (e.g., $0$ to $32767$ in the Jack language).



### **The Parser (Syntactic Analysis)**
The Parser takes the flat list of tokens generated by the Tokenizer and organizes them into a hierarchical structure according to the formal grammar of the language.

*   **Structure Building**: It creates a **Parse Tree** (often represented as an XML file in this assignment) that reflects the nested nature of the code, such as statements inside subroutines, and subroutines inside classes.
*   **Grammar Enforcement**: It ensures the tokens follow strict rules, such as verifying that an `if` statement is followed by an expression in parentheses.
*   **Look-Ahead (LL(2))**: In complex cases, the parser "peeks" at the next one or two tokens to determine the correct grammatical path, such as distinguishing between a variable name and a subroutine call[cite: 1].

## 🚀 Getting Started

### 1. Prerequisites
You must have the **Rust toolchain** installed on your device.
* **Linux/macOS**: 
  ```bash
  curl --proto '=https' --tlsv1.2 -sSf [https://sh.rustup.rs](https://sh.rustup.rs) | sh
  ```
* **Windows**: Download and run `rustup-init.exe` from [rustup.rs](https://rustup.rs/).

### 2. Clone/Navigate to the Project
Navigate to your project root in the terminal:
```bash
cd ~/compiler
```

### 3. File Structure
Ensure your directory matches this systematic structure for successful compilation:
```text
compiler/
├── Cargo.toml       # Build configuration
├── Main.jack        # Input Jack source code
└── src/
    ├── main.rs      # Orchestrates the process
    ├── tokenizer.rs # Lexical analysis logic
    └── parser.rs    # Syntactic analysis logic
```

---

## 🛠️ Usage

### 1. Changing the Input (`Main.jack`)
To compile different Jack code, replace the contents of `Main.jack` in the root directory.

**Example:**
```jack
class Main {
    function void main() {
        let x = 10;
        do Output.printInt(x);
        return;
    }
}
```

### 2. Compile and Run
Use **Cargo** to build and execute the compiler in one step:
```bash
cargo run
```

### 3. View the Outputs
The compiler generates two systematic files in the root directory:
* **`MainT.xml`**: A flat list of all tokens (keywords, symbols, identifiers, constants).
* **`Main.xml`**: A nested, indented parse tree reflecting the grammar structure.

---

## 📝 Systematic Features

* **LL(2) Parsing**: The parser uses look-ahead logic to distinguish between plain identifiers, array entries, and subroutine calls.
* **XML Escaping**: Automatically handles special characters like `&`, `<`, and `>` to ensure valid XML output.
* **Recursive Descent**: Implements a dedicated method for every rule in the Jack grammar specification.
